### 1. Spring Batch: 다중 Job 실행 에러 (`2026-02-18-batch-multiple-jobs.md`)

```markdown
# [Spring Batch] 다중 Job 실행 시 IllegalStateException 에러

**Date:** 2026-02-18
**Category:** Spring Batch

## 🚨 Error Log
```text
Caused by: java.lang.IllegalStateException: Job name must be specified in case of multiple jobs

```

## 🔍 Cause

Spring Batch 컨텍스트에 여러 개의 Job이 등록되어 있으나, 애플리케이션 실행 시 어떤 Job을 돌려야 할지 명시되지 않아서 발생. (애플리케이션이 무엇을 실행해야 할지 모르는 상태)

## 💡 Solution

`application.properties` 또는 `application.yml` 파일에 실행할 Job 이름을 명시적으로 정의한다.

## 💻 Code / Config

```yaml
# application.yml
spring:
  batch:
    job:
      name: 실행할_Job_이름

```

```

### 2. JPA/Hibernate: JPQL 엔티티 인식 에러 (`2026-02-18-jpa-unknown-entity.md`)

```markdown
# [JPA] UnknownEntityException: Could not resolve root entity

**Date:** 2026-02-18
**Category:** JPA / Hibernate

## 🚨 Error Log
```text
java.lang.IllegalArgumentException: org.hibernate.query.sqm.UnknownEntityException: Could not resolve root entity 'orders'

```

## 🔍 Cause

JPA(Hibernate)에서 사용하는 JPQL 작성 시, 자바 엔티티 클래스 이름이 아닌 데이터베이스의 테이블 이름(`orders`)을 기준으로 쿼리를 작성하여 엔티티를 찾지 못함. JPQL은 데이터베이스 테이블명이 아닌 자바 엔티티 클래스명을 기준으로 하며, 대소문자를 엄격하게 구분함.

## 💡 Solution

JPQL 쿼리 작성 시, 테이블 이름 대신 **자바 엔티티 클래스 이름**을 정확한 대소문자로 맞추어 작성한다.

## 💻 Code

**❌ Before (Error)**

```java
// 테이블명을 그대로 사용해서 에러 발생
String jpql = "SELECT o FROM orders o";

```

**✅ After (Resolved)**

```java
// 엔티티 클래스명(Order)을 정확히 사용
String jpql = "SELECT o FROM Order o";

```

```
